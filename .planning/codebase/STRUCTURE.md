# Codebase Structure

**Analysis Date:** 2026-05-20

## Directory Layout

```
langchain-project/
├── inputPDF/          # Source PDF documents to process
├── output/            # Extracted text and OCR results
│   └── images/        # Per-page images extracted for OCR
├── faiss_index/       # FAISS vector index (binary, gitignored)
├── __pycache__/       # Python bytecode cache (gitignored)
├── .planning/         # GSD project planning directory
├── main.py            # Entry point — orchestrator + REPL
├── extract_text.py    # PDF text/OCR extraction module
├── text_chunker.py    # Text splitting + embedding module
├── requirements.txt   # Python dependencies
├── .env               # API keys (gitignored)
├── .gitignore         # Git exclusion rules
└── .git/              # Git repository metadata
```

## Directory Purposes

**inputPDF/:**
- Purpose: Place PDF documents here for processing
- Contains: `*.pdf` files
- Key files: `GeoBase_NHNC1_Data_Model_UML_EN.pdf`, `PublicWaterMassMailing.pdf`, `resume_ayushsaraf.pdf`

**output/:**
- Purpose: Extracted text and OCR results
- Contains: `extracted_text_*.txt`, `image_ocr_*.txt` (one pair per PDF)
- Subdirectories: `images/` — extracted page images for OCR

**faiss_index/:**
- Purpose: FAISS vector index files (binary)
- Gitignored — reconstructed from PDFs on changes

## Key File Locations

**Entry Points:**
- `main.py` — Application entry point (`python main.py`)

**Core Logic:**
- `extract_text.py` — PDF detection, text extraction, OCR pipeline (192 lines)
- `text_chunker.py` — Text splitting, embedding generation, FAISS persistence (130 lines)
- `main.py` — Pipeline orchestration, similarity search, LLM chat loop (112 lines)

**Configuration:**
- `requirements.txt` — Python package dependencies
- `.env` — API keys (gitignored)
- `.gitignore` — Exclusion rules

**Documentation:**
- `requirements.txt` (the only project doc)

## Naming Conventions

**Files:**
- `snake_case.py` for Python modules
- `UPPER_SNAKE_CASE` for configuration constants
- `extracted_text_{pdf_stem}.txt` — extracted text output
- `image_ocr_{pdf_stem}.txt` — OCR output

**Functions:**
- `snake_case()` for all functions (Python standard)
- `main()` for module entry points

## Where to Add New Code

**New Document Source:**
- Add parser: New `.py` module at project root
- Add output format: Extend `output/` handling

**New LLM/Embedding Provider:**
- Add connector: New integration in existing modules or new module
- Config: Add constants + `.env` vars

**New Feature:**
- Script-level: Add function to existing module or create new `.py` file
- Wire through `main()` orchestrator

## Special Directories

**__pycache__/:**
- Purpose: Python bytecode cache
- Source: Auto-generated by Python interpreter
- Committed: No (in .gitignore)

**faiss_index/:**
- Purpose: Binary FAISS vector store
- Source: Generated by text_chunker.py
- Committed: No (in .gitignore)
